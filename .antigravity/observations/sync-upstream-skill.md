# sync-upstream-skill への知見 (Observations)

## 概要
移植元リポジトリの更新に追随する際、ロジックのマージだけでなく、関連ファイルの増減や構成変更を見落とさないためのガードレールです。

## 物理的なガードレール

### 1. 構成変更（ファイル増減）の完全な把握
- **RED**: `git diff --stat` で統計を見るだけで満足し、スキル配下（`skills/`）以外の変更を「関連なし」と即断して無視してしまった。
- **GREEN**: `git diff --name-status` を使用し、全ファイルの増減・移動を一行ずつ精査せよ。
- **指示**: 同期対象のスキルに関連するファイル（`agents/` 内のエージェント定義、`tests/` 内の検証スクリプト等）が削除・移動・追加されている場合、必ずそれらも同期対象に含めなければならない。

### 2. DoD（完了の定義）の物理的強制
- **RED**: 作業が一段落した高揚感から、DoD に含まれる「コードレビュー」を失念して完了を宣言した。
- **GREEN**: 完了宣言（DoD の達成主張）を行う前に、必ずスキルの `DoD` セクションを物理的に再読み込みし、各項目をチェックリストとして一つずつ検証せよ。
- **指示**: 特に「コードレビュー」や「物理的な検証」は、最後に回すと省略されやすいため、それらが完了するまで「完了の定義」という言葉を安易に使用してはならない。

## 関連規律
- `sync-upstream-skill`: 本知見を同期プロセスの Inventory フェーズおよび Finalization フェーズに適用せよ。
