# リリース完全性のための「コアファイル」意識 (Core File Discipline)

## 1. 概要
本プロジェクト（スキル移植工場）において、エクステンションの価値はスキル、ロジック、および憲法（規律）の整合性によって担保される。リリース（マージ）前のチェックにおいて、これらの「コアファイル」に変更があった場合は、必ずバージョン番号を更新しなければならない。

## 2. 実質的なロジック変更と見なす対象 (Core Files)
- **`scripts/`**: スキルの動作を支える制御ロジック。
- **`agents/`**: サブエージェントの思考を規定するプロンプト。
- **`GEMINI.md`**: エクステンション全体の行動憲法（システムプロンプト）。

## 3. 判断の基準
- **URL インストールの影響**: `GEMINI.md` や `agents/` の変更は、世界中のユーザーがエクステンションを使用した際の AI の振る舞いに直接影響を与える。
- **バージョンアップの責任**: コアファイルの一部でも変更されたなら、たとえそれが微調整であっても、ユーザーにアップデートを促すためにバージョン番号を上げよ（パッチ、またはマイナー）。
- **DoD の厳格化**: `finish_check.mjs` がパスしたとしても、AI は自律的に「今回の変更はユーザーの体験に影響を与えるか？」を自問し、必要ならバージョンアップを提案せよ。

## 4. 禁止事項
- **DoD 違反の正当化**: 「小さな変更だから」という理由で、コアファイルの変更時にバージョンアップを省略してはならない。
