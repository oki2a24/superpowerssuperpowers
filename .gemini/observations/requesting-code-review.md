# 知見：requesting-code-review (Gemini固有)

## 概要
`requesting-code-review` スキルを Gemini CLI 環境で運用する際の、物理的なタイミングとルールの厳格化。

## 物理的タイミングの強制
- **コミット直前の最終ゲート**: 開発ブランチへの `git commit` を行う「直前」に、必ず `code-reviewer` エージェントを派遣し、実装の最終確認を行わなければならない。
- **実証テスト後のレビュー**: 物理的な検証（テストパス等）が完了した「後」に、その結果を含めてレビューを依頼せよ。
- **指摘の反映**: 指摘事項（タイポ等）を修正した後、再度コミットする前にもう一度レビューが必要かを自己判断せよ。軽微な修正であっても、DoD の達成を物理的に確認しなければならない。

## テンプレートの使用
- **code-reviewer.md**: 常に `skills/requesting-code-review/code-reviewer.md` テンプレートを使用せよ。
- **プレースホルダの厳密性**: `{WHAT_WAS_IMPLEMENTED}`, `{PLAN_OR_REQUIREMENTS}` 等のプレースホルダを漏れなく埋め、レビュアーに十分なコンテキストを提供せよ。
- **用語適応**: テンプレート内の用語（「派遣」「サブセッション」等）を尊重し、Gemini CLI のモデルに合わせた対話を行え。
