# 知見： ドキュメンテーション (README & Skills)

## 概要
README やスキル定義は、ユーザーだけでなく **AI エージェント自身が自分の能力を正しく認識し、活用するための「設計図」** である。

## 知見 (Observations)
- **AI レゾナンスの強化**: README には必ず「各スキルの目的 (Intent)」と「具体的なプロンプト例」を含めよ。これにより、エージェントは自分がいつ、どのスキルを使うべきかを自己発見できる。
- **UX 最優先のクイックスタート**: インストール後、ユーザーが「最初の一歩」を迷わないよう、コピー＆ペースト可能な 3 ステップ構成（Install -> Check -> Run）を標準とせよ。
- **セキュリティの明文化**: ローカル実行、外部通信の有無、使用言語（Node.js）を明示し、ユーザーに技術的な透明性と安心感を提供せよ。
- **リスペクトの物理的配置**: 移植元の謝辞は、単なるテキストではなく「IMPORTANT」ブロックや専用セクションで強調し、エンジニアリング・コミュニティへの敬意を形にせよ。
