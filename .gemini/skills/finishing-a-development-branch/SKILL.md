# 開発ブランチの完了

## 概要

このスキルは、実装作業の完了をガイドし、明確なオプションを提示して、選択されたワークフローを処理します。

**コア原則:** テストの検証 → オプションの提示 → 選択の実行 → クリーンアップ。

## プロセス

### 1. テストの検証

オプションを提示する前に、**必ずテストがパスすることを確認**します。

*   **AIエージェントへの指示**: プロジェクト固有のテストコマンドをユーザーに尋ねてください。（例: `npm test`, `cargo test`, `pytest`, `go test ./...`）
*   ユーザーから提供されたコマンドを使って、`run_shell_command`でテストスイートを実行します。

**テストが失敗した場合:**
*   「テストが失敗しました（<N>件）。続行する前に修正が必要です。」とユーザーに報告し、失敗の詳細を表示します。
*   「テストがパスするまで、マージ/PRに進むことはできません。」と伝えて停止し、ユーザーの指示を待ちます。

**テストがパスした場合:**
*   「テストはすべてパスしました。」と報告し、次のステップに進むかユーザーに確認してください。

### 2. ベースブランチの決定

*   **AIエージェントへの指示**: `git merge-base HEAD main 2>/dev/null`および`git merge-base HEAD master 2>/dev/null`を実行し、ベースブランチのヒントを取得します。
*   `main`または`master`がベースブランチとして検出されない場合、またはユーザーに確認が必要な場合、「このブランチは`main`から分岐しましたが、正しいですか？」のように尋ねて、ベースブランチを確定します。

### 3. オプションの提示

実装が完了したことをユーザーに伝え、以下の4つのオプションを**簡潔に**提示します。

```
実装が完了しました。何をしますか？

1. ローカルで<ベースブランチ>にマージする
2. プッシュしてプルリクエストを作成する
3. ブランチを現状維持する（後で自分で処理します）
4. この作業を破棄する

どのオプションを選びますか？
```

*   **AIエージェントへの指示**: 上記のオプションを提示し、ユーザーの選択を取得してください。

### 4. 選択の実行

ユーザーの選択に応じて、以下のアクションを実行します。

*   **オプション1: ローカルでマージ**:
    *   **AIエージェントへの指示**: `git checkout <ベースブランチ>`を実行してベースブランチに切り替えます。
    *   **AIエージェントへの指示**: `git pull`を実行して最新の変更を取得します。
    *   **AIエージェントへの指示**: `git merge <フィーチャーブランチ名>`を実行してフィーチャーブランチをマージします。
    *   マージ後、再度テストを実行し、問題がないことを確認します。
    *   **AIエージェントへの指示**: テストがパスしたら、`git branch -d <フィーチャーブランチ名>`を実行してフィーチャーブランチを削除し、次のステップに進むかユーザーに確認してください。

*   **オプション2: プッシュしてプルリクエストを作成**:
    *   **AIエージェントへの指示**: `git push -u origin <フィーチャーブランチ名>`を実行してブランチをプッシュします。
    *   **AIエージェントへの指示**: プルリクエストのタイトルと概要をユーザーに尋ね、GitHub CLI (`gh pr create`) を使ってプルリクエストを作成します。GitHub CLIがインストールされていない場合や利用できない場合は、ユーザーに手動で作成するよう促します。
    *   **AIエージェントへの指示**: プルリクエスト作成後、次のステップに進むかユーザーに確認してください。

*   **オプション3: ブランチを現状維持する**:
    *   「ブランチ`<フィーチャーブランチ名>`は現状維持されます。ワークツリー（存在する場合）も保持されます。」とユーザーに報告し、次のステップに進むか確認してください。

*   **オプション4: この作業を破棄する**:
    *   **AIエージェントへの指示**: 破棄の前に、以下の内容をユーザーに伝え、正確に「discard」と入力して確認するよう促してください。
        ```
        これは永続的に削除されます:
        - ブランチ <フィーチャーブランチ名>
        - すべてのコミット: <コミットリスト>
        - ワークツリー（存在する場合）

        確認のために 'discard' と入力してください。
        ```
    *   ユーザーが「discard」と入力して確認した場合、`git checkout <ベースブランチ>`、`git branch -D <フィーチャーブランチ名>`を実行してブランチを削除します。
    *   **AIエージェントへの指示**: 削除後、次のステップに進むかユーザーに確認してください。

### ステップ 5: ワークツリーのクリーンアップ

**オプション 1, 2, 4 の場合:**

*   **AIエージェントへの指示**: 以下のコマンドで、現在のブランチがワークツリー（メインワークスペース以外）にあるか確認します。
```bash
git worktree list | grep $(git branch --show-current)
```

ワークツリー内にいる場合、それを削除します。
*   **AIエージェントへの指示**: **重要**: ワークツリーの削除コマンドは、必ずプロジェクトルート（メインワークスペース）のコンテキストで実行してください（`run_shell_command` の `dir_path` を指定しない）。これにより、自分自身のディレクトリを削除しようとして発生するエラーを回避します。
```bash
git worktree remove <ワークツリーパス>
```

*   **ローカルの補足 (Gemini固有)**: `.gemini/tasks/TODO-<フィーチャーブランチ名>.md` が存在する場合、役割を終えたタスク管理ファイルとして削除します。

**オプション 3 の場合:** ワークツリーを保持します。

## 参考情報

### クイックリファレンス

| オプション | マージ | プッシュ | ワークツリーを保持 | ブランチのクリーンアップ |
|--------|-------|------|---------------|----------------|
| 1. ローカルマージ | ✓ | - | - | ✓ |
| 2. PR作成 | - | ✓ | ✓ | - |
| 3. 現状維持 | - | - | ✓ | - |
| 4. 破棄 | - | - | - | ✓ (強制) |

### よくある間違い

*   **テスト検証のスキップ**:
    *   **問題:** 壊れたコードをマージしたり、失敗するPRを作成したりする。
    *   **修正:** オプションを提示する前に常にテストを検証する。
*   **オープンエンドな質問**:
    *   **問題:** 「次は何をすべきですか？」→ 曖昧。
    *   **修正:** 正確に構造化された4つのオプションを提示する。
*   **ワークツリーの自動クリーンアップ**:
    *   **問題:** 必要になる可能性があるワークツリーを削除してしまう (オプション2, 3)。
    *   **修正:** オプション1と4の場合にのみクリーンアップする。
*   **破棄の確認なし**:
    *   **問題:** 誤って作業を削除してしまう。
    *   **修正:** オプション4では「discard」と入力して確認することを求める。

### 危険な兆候

**決してしないこと:**
*   テストが失敗したまま続行する。
*   マージ後にテスト結果を検証せずにマージする。
*   確認なしで作業を削除する。
*   明示的な要求なしに強制プッシュする。

**常にすること:**
*   オプションを提示する前にテストを検証する。
*   正確に4つのオプションを提示する。
*   オプション4ではタイプ入力による確認を得る。
*   ワークツリーのクリーンアップはオプション1と4の場合にのみ行う。

## 統合

**呼び出し元:**
*   `subagent-driven-development` (ステップ7) - すべてのタスク完了後。
*   `executing-plans` (ステップ5) - すべてのバッチ完了後。

**連携先:**
*   `using-git-worktrees` - そのスキルによって作成されたワークツリーをクリーンアップします。
