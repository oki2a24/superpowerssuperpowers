# Upstream 同期規律 (Upstream Sync)

## 概要
上流リポジトリの最新状態をチェックし、物理的なファイルの更新（サブモジュールの同期）から、日本語スキルへのロジック注入までをワンストップで実行します。

## いつ使用するか
- 上流リポジトリに新しい改善があるか確認し、同期したい時。
- 「上流に追随して」「最新化して」などの指示を受けた時。

## コア・プロトコル

### 0. 物理的な更新 (Phase: Fetch)
- `superpowers-original` ディレクトリにて、上流（`obra/superpowers`）の最新コミットを物理的に取得（`git fetch` および `git checkout` 等）する。
- 変更がない場合はその旨を報告し、同期プロセスを終了する。

### 1. 差分スキャン (Phase: Inventory)
- `docs/superpowers_ports.md` を唯一の真実 (SSOT) とし、そこに記載されているスキルのみを対象とする。
- **除外**: 本PJオリジナルのスキル（移植履歴にないもの）は一切触れない。
- `SKILL.md` だけでなく、関連するエージェント定義やスクリプトも対象に含める。

### 2. ロジック抽出と提案 (Phase: Proposal)
- 英語の差分から以下の「エッセンス」を抽出し、日本語版への適用を提案する：
    - ガードレール (Gates/MUST/NEVER)
    - 具体的手順 (Steps/Checklists)
    - アンチパターン (Anti-patterns)
    - 設計思想 (Principles/Insights)
- **鉄則**: 日本語表現を英語で上書きしてはならない。ロジックのみを注入（マージ）せよ。

### 3. 適応の規律 (Unified SOP)
- **必須サブスキル**: 新規追加や大規模な更新には `port-superpowers-skill` の分析・ドラフトプロセスを再利用せよ。

## アンチパターン
- **翻訳の書き戻し**: 日本語化済みのスキルを、上流の英語でそのまま上書きすること。
- **オリジナルスキルの破壊**: 移植履歴にない本プロジェクト独自のスキルを変更・削除すること。
- **検証なき同期**: 同期後に `code-reviewer` による日本語品質とロジックのチェックをスキップすること。

## 完了の定義 (DoD)
- `docs/superpowers_ports.md` の参照コミットハッシュが更新されている。
- `code-reviewer` による承認（Approved）を得ている。
- 物理的な動作検証（`activate_skill` でのロード確認）を完了している。
