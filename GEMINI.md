# Gemini CLI 運用記憶と学習記録 (SSOT)

本ファイルは、Gemini CLI エージェントがこのプロジェクトにおいて遵守すべき行動規範、技術的決定事項、および実戦から得た知見を永続的に記録した唯一の真実のソース（SSOT）である。

## 1. AIエージェントの行動規律 (Core Mandates)

### Git 操作の徹底 (Git Discipline)
- **ブランチの保護**: 開発着手前に必ず現在のブランチを確認すること。`main` ブランチへの直接的な変更やコミットは、明示的なユーザー指示がない限り厳禁とする。
- **原子的なコミット (Atomic Commits)**: TDD の各タスク（GREEN 達成時）ごとに、その成果を即座にコミットすること。コミット漏れは履歴の劣化を招くため、タスク完了の必須条件とする。

### 知見のコード内還元
- **試行錯誤の記録**: 実装の過程で生じた複雑なロジック、解決済みの曖昧さ、または特殊な制約への対処については、その背景をソースコード（スクリプトおよびテスト）内のコメントとして厚く残すこと。「動くコード」以上に「なぜそのコードになったか」という文脈を重視する。

### 進捗管理と同期
- **ロードマップの提示**: 複数ステップにわたる大規模タスクでは、開始時に「ステップごとの使用スキル」を含むロードマップを提示すること。
- **現在地の明示**: 各タスクの変わり目（コミット直後など）には、必ず「現在地：Task M / 次：Task M+1」を報告し、ユーザーとコンテキストを同期すること。

## 2. 開発・テストワークフロー

### 作業の分離の原則
- **フィーチャー開発**: 永続的な変更は必ず専用のフィーチャーブランチで行う。
- **検証環境の隔離**: 実装環境を汚染せず、スキルの再起動等を確実に行うため、テストや検証は `using-git-worktrees` を使用した独立したワークツリーで実行することを原則とする。

### 技術選定と低オーバーヘッド
- **標準ライブラリの優先**: 環境構築のコストを最小化しポータビリティを高めるため、Python および Node.js においては標準ライブラリのみで完結させる実装を最優先する。外部パッケージ（pip/npm等）への依存は、その必要性を厳格に評価した上で導入すること。

### セッション管理の定義
- **セッション・リフレッシュ**: 履歴を破棄してコンテキストを節約する行為。再開用プロンプトを必須とする。
- **スキルの認識（再起動）**: `gemini --resume latest` を使い、履歴を保持したまま新しいスキルを読み込ませる行為。

## 3. セッション間連携 (GPAC)

### プロトコルの遵守
- **SSOT の場所**: サブセッションのデータは `~/.gemini/sub-sessions/` に集約し、プロジェクト横断的な一貫性を維持する。
- **Workspace Boundary の克服**: ワークスペース外（境界外）のディレクトリ操作には、`cat`/`cp` 等の低レイヤーなシェルコマンドではなく、`gemini-sub` が提供する専用 API (`show-task`, `show-report` 等) を優先して使用すること。
- **Handoff (引き渡し) 方式**: 書き込み制限を回避するため、ワークスペース内で「下書き」を仕上げ、スクリプトがそれを検証・正規化してグローバル領域へ移動させるフローを標準とする。
