# SuperpowersSuperpowers 🚀

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Node.js](https://img.shields.io/badge/Node.js-v20+-green.svg)](https://nodejs.org/)

**Gemini CLI のための高精度インテリジェンス & スキル拡張**

`SuperpowersSuperpowers` は、Gemini CLI に「高度な思考プロセス」と「自律的な自己進化」を統合する、プロフェッショナル仕様の拡張機能です。
業界標準のエンジニアリング規律（移植軸）と、長期的な経験を資産化する独自知能（知能軸）を融合させることで、単なるツールを超えた「成長し続けるエンジニアリング・パートナー」を定義します。

---

## 🏛 二軸のインテリジェンス・アーキテクチャ (Dual-Axis Architecture)

本プロジェクトは、互いに相乗効果を生み出す 2 つの柱によって構成されています。

### 1. Inherited Foundations: 信頼の基盤
[obra/superpowers](https://github.com/obra/superpowers) の優れた設計思想を Gemini CLI へ忠実に移植。
*   **規律の物理的強制**: TDD（テスト駆動開発）、体系的なデバッグ、および構造化された実装計画。
*   **安全な実行環境**: Git Worktrees による環境隔離と、サブエージェントへの確実な任務委譲。
*   **信頼性**: すべての移植スキルは Gemini CLI のランタイムに最適化され、予測可能で高品質な成果を保証します。

### 2. Evolutionary Intelligence: 自律する知能
本プロジェクト独自の、長期的なコンテキスト管理と「英知」の蓄積に特化した知能レイヤー。
*   **知見の蒸留 (Distillation)**: 実戦での学びを「知見の地層 (Observation Strata)」として抽出し、ポータブルな資産へ変換。
*   **継続性 (Continuity)**: 大規模セッションにおける「無損失」なコンテキスト引き継ぎ（Handoff）。
*   **協調 (Coordination)**: GPAC プロトコルに基づく、サブセッションとの高度な任務調整と同期。

## 🚀 クイックスタート (Quick Start)

インストールから最初のスキル起動まで、わずか 3 ステップです。

### 1. インストール
```bash
gemini extensions install https://github.com/oki2a24/superpowerssuperpowers
```

### 2. スキルの確認
インストール後、エージェントにこう問いかけてください：
> 「導入された superpowers スキルのリストを見せて」

### 3. 最初のスキル起動
長期的なタスクを開始する前に、進捗を管理するスキルを起動するのがおすすめです：
> 「`roadmap-management` スキルを起動して、今回の作業の計画を立てて」

## 🏛 アーキテクチャ & システム・エフェクト (Architecture & Side Effects)

`SuperpowersSuperpowers` は、ユーザーの環境に「英知の地層」を形成します。以下の図解は、スキル実行時に AI がどのようにコンテキストを合成し、どのディレクトリに影響を与えるかを示しています。

### 1. 知見の地層と優先順位 (The Strata)

AI エージェントは、実行時に以下の 4 つの層を順にスキャンし、最新の知見を動的に合成（上書き）します。右側（L4）に近いほど優先度が高くなります。

```text
[ 低優先 (基盤) ]                                                                                  [ 高優先 (最新) ]

     L1: Core (Skills)            L2: Extension (Shared)         L3: Personal (Global)        L4: Project (Local)
 [Extension Root]/skills      [Extension Root]/observations    ~/.gemini/observations/      ./.gemini/observations/
           |                            |                             |                            |
           +----------------------------+-----------------------------+----------------------------+
                                        |
                                        v
                               [ AI Execution Context ]
```

### 2. ファイルシステムへの影響 (Filesystem Side Effects)

本拡張機能は、ユーザーの「英知」を永続化するために以下のディレクトリを使用します。これらはすべてプレーンな Markdown 形式であり、ユーザーの好みに応じてバージョン管理に含めることも可能です。

| パス (Path) | 役割 (Role) | バージョン管理の推奨 |
| :--- | :--- | :--- |
| **`./.gemini/observations/`** | **L4: Project Local** | **[任意]** プロジェクト固有の知見やルール。チームで規律を共有したい場合は Git に含め、個人用とする場合は `.gitignore` に追加してください。 |
| **`~/.gemini/observations/`** | **L3: Personal Global** | **[通常は不要]** ユーザー個人の好みや、全プロジェクト共通の規律。通常は OS のユーザーディレクトリに保持されます。 |

> [!TIP]  
> **柔軟な運用**:  
> 本拡張機能は、プロジェクトのルートディレクトリを汚すことなく、`.gemini/` フォルダ内に設定や知見を集約します。これにより、メインのソースコードの構成を維持したまま、AI エージェントの知能を段階的に強化できます。

### 3. GPAC プロトコル (Gemini Peer-Agent Coordination)

親セッションとサブセッションの間で任務を確実に同期するための独自通信プロトコルです。
*   **任務の委譲 (Spawn)**: コンテキストを適切に制限し、サブセッションへ任務を物理的に分離して委譲。
*   **結果の帰還 (Import)**: サブセッションでの成果を構造化されたレポートとして親セッションへ統合。
*   **整合性の維持**: セッションを跨いでも「現在地」を見失わず、大規模開発の迷走を物理的に防ぎます。

## 🔒 セキュリティ & プライバシー (Security & Privacy)

プロフェッショナルなツールとして、ユーザーの資産とプライバシーの保護を最優先事項として定義しています。

*   **100% ローカル実行**: 推論（Gemini API 呼び出し）以外のすべての操作、スクリプト、知見の保存はローカル環境で完結します。
*   **透明な英知 (Transparent Knowledge)**: 採取された知見はすべて Markdown 形式で保存されます。暗号化や難読化は一切なく、ユーザーはいつでも自らの英知を監査・修正できます。
*   **最小権限の原則**: ツールはプロジェクトディレクトリおよび `~/.gemini/` 内の規定のパス以外の領域には干渉しません。

## 📂 ディレクトリ構造

- **`skills/`**: コア・スキル (移植済み & オリジナル)
- **`observations/`**: 共有知見レイヤー (L2: Extension Shared)
- **`.gemini/skills/`**: 開発中のメタ・スキル
- **`.gemini/observations/`**: プロジェクト固有の知見 (L4: Project Local)
- **`agents/`**: サブエージェントのプロンプト定義
- **`scripts/`**: 運用支援ツール (`todo.mjs` 等)
- **`docs/`**: デザインドキュメント、実装計画、移植記録
