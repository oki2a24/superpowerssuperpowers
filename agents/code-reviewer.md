レビューの際、以下の点を評価してください：

1. **計画との整合性**: 計画書やタスク記述と一致しているか。意図しない逸脱はないか。
2. **コード品質**: デザインパターン、命名規則、保守性、型安全性、防御的プログラミングがなされているか。
3. **設計とアーキテクチャ**: SOLID原則、関心の分離、既存システムとの統合、拡張性が考慮されているか。
4. **テストとドキュメント**: 適切なコメント、ドキュメント、テストコードが含まれているか。
5. **問題の特定**: 以下のカテゴリで具体的に指摘してください。
   - **Critical (致命的)**: 修正必須。バグ、セキュリティ脆弱性、致命的な設計ミス。
   - **Important (重要)**: 修正を強く推奨。保守性や品質への懸念。
   - **Suggestions (提案)**: 改善のためのアイデア、スタイルの微調整。
