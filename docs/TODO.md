## 4. 今後の課題と改善
- [ ] **`gemini-sub` へのテンプレート生成機能の追加と責務の明確化**:
    - **背景・経緯**:
        1. 本プロジェクトでは、タスク委譲の自動化のため、補助スクリプト `gemini-sub.mjs` が先に開発された。
        2. その後、AIエージェントがそのスクリプトを自律的に使用するための「使い方」として `session-coordination` スキルが作成された。
        3. しかし、スキルを作成する過程で、エージェントは `spawn` や `report` に必要な下書き（テンプレート）の準備を「**AI（自分）が手作業で行うべきタスク**」と誤って解釈・実装してしまった。
        4. その結果、ツール開発者の「自動化された部品」という意図と、ツール利用者の「手作業を含む手順」という解釈の間にギャップが生まれ、E2Eテストでのフォーマットエラー頻発に繋がった。
    - **教訓**: スキルとその補助スクリプトは、常にペアで設計されるべきである。特に、ツール（補助スクリプト）の提供者は、その「使い方」まで含めて、可能な限り自動化された形で（＝コマンドオプションやサブコマンドとして）提供し、「利用者の手作業」に依存する部分を極限まで減らすべきである。
    - **要件**:
        1. `node scripts/gemini_sub.mjs new-task <filename.md>` コマンドを追加し、正しい Frontmatter 形式の下書きをファイルに出力する。
        2. `node scripts/gemini_sub.mjs new-report <filename.md>` コマンドを追加し、正しい Frontmatter 形式の報告書下書きをファイルに出力する。
        3. `session-coordination` スキルを修正し、下書き作成のステップを、これらの新コマンドの実行に置き換える。
        4. これにより、`gemini-sub.mjs` を、使い方まで含めて自己完結した「パターン2」の補助スクリプトへと昇格させる。
