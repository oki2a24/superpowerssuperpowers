        - **背景**: Python 版のテストでは実際の `.gemini/tasks/` を使用していたため、テスト実行がユーザーのタスクファイルを汚染するリスクがあった。

        - **背景**: Node.js の ESM において、`process.argv[1]` のパス形式が起動方法（相対/絶対/シンボリックリンク）によって変動し、Python の `if __name__ == "__main__":` 相当の判定が不安定になる事象が発生した。
        - **要件**: `url.fileURLToPath(import.meta.url)` と `path.resolve(process.argv[1])` を比較する、環境に依存しない堅牢な直接実行判定ロジックを確立し、全スクリプトに適用せよ。

        - **背景**: 移植過程で、テスト互換性のための「厳格なメッセージ一致」と、ユーザー向けの「クリーンな出力（スタックトレース抑制）」の両立が求められた。
        - **要件**: 既知のエラー（YamlSyntaxError 等）を捕捉して 1 行で出力する標準的な `try...catch` パターンを `main` 関数に実装し、CLI としての品位を高めよ。
