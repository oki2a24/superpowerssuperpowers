- [ ] **Node.js 移行後の品質強化 (Phase 2)**:
    - [ ] **`scripts/todo.mjs` の環境変数化とテストの隔離**:
        - **背景**: Python 版のテストでは実際の `.gemini/tasks/` を使用していたため、テスト実行がユーザーのタスクファイルを汚染するリスクがあった。
        - **要件**: `TASK_DIR` を環境変数 `GEMINI_TASK_DIR` で上書き可能にする。テストコードでは `tmpdir` を割り当てて環境変数を注入し、物理的な副作用を完全に隔離せよ。
    - [ ] **ESM 直接実行判定の標準化**:
        - **背景**: Node.js の ESM において、`process.argv[1]` のパス形式が起動方法（相対/絶対/シンボリックリンク）によって変動し、Python の `if __name__ == "__main__":` 相当の判定が不安定になる事象が発生した。
        - **要件**: `url.fileURLToPath(import.meta.url)` と `path.resolve(process.argv[1])` を比較する、環境に依存しない堅牢な直接実行判定ロジックを確立し、全スクリプトに適用せよ。
    - [ ] **CLI エラーハンドリングの洗練**:
        - **背景**: 移植過程で、テスト互換性のための「厳格なメッセージ一致」と、ユーザー向けの「クリーンな出力（スタックトレース抑制）」の両立が求められた。
        - **要件**: 既知のエラー（YamlSyntaxError 等）を捕捉して 1 行で出力する標準的な `try...catch` パターンを `main` 関数に実装し、CLI としての品位を高めよ。
    - [ ] **`gemini-sub.mjs` の `work_dir` 解析の自動化**:
        - **背景**: 移行中、メインワークスペースからワークツリーを指定して `spawn` した際、生成された BOOT コマンドの `cd` 先が `os.getcwd()`（実行元）に固定され、不整合が生じる事象が発生した。
        - **要件**: YAML Frontmatter 内の `work_dir` を抽出し、`launch_session` に渡す作業ディレクトリとして優先的に使用するロジックを実装せよ。
    - [ ] **バリデーションエラー時のユーザー補助強化**:
        - **背景**: `parent_task_tag` 等の必須キーが欠落した際、単に「Missing required key」と表示されるだけで、正しい形式を思い出すための負担が大きかった。
        - **要件**: 必須キーの欠落などのバリデーションエラー発生時に、期待される Frontmatter のテンプレートやヘルプを表示する機能を `gemini-sub.mjs` に追加せよ。
    - [ ] **`task.md` の構造拡張と `required_skills` の導入**:
        - **背景**: サブセッションへの指示において、使用すべきスキルの明示（`required_skills`）が Frontmatter に欠落していた。
        - **要件**: `required_skills` キーを必須（または任意）で追加し、エージェントが初動で迷わず `activate_skill` を行えるようにせよ。
    - [ ] **`gemini-sub.mjs` のプロンプト設計を「抽象化 I/O」へ転換**:
        - **背景**: `spawn` 時のプロンプトが `cat <path>` を指示していたため、エージェントが `show-task` コマンドという抽象化された手段を無視して物理パスを操作していた。
        - **要件**: `launch_session` が生成するプロンプトを `python3 scripts/gemini_sub.mjs show-task <id>` を推奨するように修正せよ。
    - [ ] **テストコードと JSDoc の品質基準の確立**:
        - **背景**: テストタイトルの日本語化は有効だったが、タイトルと詳細コメントの役割分担が未整理。また、スクリプトの関数のドキュメント（JSDoc）も不足している。
        - **要件**: `test('日本語タイトル', ...)` を標準化し、背景は Docstring に記述。また全関数に `@param`, `@returns` を含む JSDoc を付与せよ。
    - [ ] **`todo.mjs` の CLI 仕様の再定義と改善**:
        - **背景**: サブセッション中のタスク管理において、エージェントが `todo` の操作（開始・終了・ID指定）に迷う場面があった。
        - **要件**: 人間とAIの両方にとって最もミスの少ない `todo` の CLI 仕様を再定義し、実装を洗練させよ。
