# AIエージェント自己改善プロセスの設計 (Self-Improvement Design)

## 1. 目的
AIエージェントが、実戦で得られた知見を自律的かつ安全に指示書（SKILL.md）へ反映し、自身のパフォーマンスを継続的に向上させる仕組みを構築する。

## 2. コア原則
- **移植元の尊重 (Upstream Integrity)**: 移植元スキル（obra/superpowers）の論理構造を破壊せず、将来のアップデート追随を可能にする。
- **物理的・論理的分離 (Hybrid Layer)**: 改善点はラベルを用いて明確に分離し、いつでも一括削除（リセット）可能にする。
- **説明責任 (Traceability)**: すべての改善には理由と日付を付記し、`GEMINI.md` に履歴を残す。

## 3. 改善のワークフロー

### 3.1. トリガー (Triggers)
- **受動的トリガー**: `finishing-a-development-branch` などの完了系スキルの終了時に、AIが改善案の有無を自発的に提案する。
- **能動的トリガー**: ユーザーが「今の件をスキルに反映して」等と指示した際に、即座に改善モードを起動する。

### 3.2. 分析と反映
- **分析**: 移植された `brainstorming` スキルを「改善モード」として活用し、セッション履歴から具体的な改善案を特定する。
- **反映**: ターゲットとなる `SKILL.md` の末尾に、以下の形式で追記を行う。
    ```markdown
    ## ローカル・アダプテーション (Gemini固有)
    <!-- IMPROVED_ON: YYYY-MM-DD | REASON: ... -->
    ### AIエージェントへの指示 (Gemini固有)
    - [改善された具体的な手順や注意点]
    ```

### 3.3. 運用記憶への統合
- 個別スキルの改善要約を `GEMINI.md` の「学び」セクションに自動追記する。

## 4. 保護と復元のメカニズム

### 4.1. インスタント復元 (Rollback)
- `scripts/reset_skill.py` を用意。このスクリプトは、指定されたファイルの `## ローカル・アダプテーション (Gemini固有)` 以降の記述をすべて削除し、純粋な移植版にリセットする。

### 4.2. アップデート時の自動保護
- `port-superpowers-skill` の SOP を更新。移植元からのアップデート時には、現状の「ローカル・アダプテーション」セクションをバックアップし、新バージョン適用後に再結合する手順を定義する。

## 5. 成功基準
- AIエージェントが、手動の修正を必要とせずに、過去の失敗から学習して手順を最適化できること。
- リセットスクリプトにより、一瞬で「純粋な移植版」の状態に戻せること。
- 将来の移植元アップデートにおいて、ローカルの改善点が競合やノイズにならず、容易に維持できること。
