# デザインドキュメント: reset_skill.py の Node.js への移行

## 1. 目的
Gemini CLI のスキルファイルを「ローカル・アダプテーション」セクション以前にリセットする Python スクリプト (`scripts/reset_skill.py`) を、Node.js (v25.8.0, ESM) に移植します。これにより、環境依存を減らし、ポータビリティを向上させます。

## 2. アーキテクチャとデータフロー

### 2.1 コンポーネント
- **`scripts/reset_skill.mjs`**: メインロジック。
    - `TARGET_HEADER`: `"## ローカル・アダプテーション (Gemini固有)"`
    - `resetSkillFile(filePath)`: ファイルを読み込み、指定のヘッダー以降を削除して上書き。
    - `main()`: コマンドライン引数を処理し、各ファイルに対して `resetSkillFile` を実行。
- **`tests/test_reset_skill.mjs`**: `node:test` を使用した単体テスト。

### 2.2 データフロー
1.  **入力**: `process.argv` からファイルパスのリストを取得。
2.  **読み取り**: `fs.readFileSync(filePath, 'utf8')` でファイル内容を取得。
3.  **加工**:
    - `TARGET_HEADER` のインデックスを検索。
    - 見つかった場合、そのインデックス以前をスライス。
    - 直前の空行を `trimEnd()` 相当の処理で除去。
    - 末尾に単一の改行 `\n` を付与。
4.  **出力**: 内容に変更があった場合のみ `fs.writeFileSync(filePath, content)` で上書き。
5.  **戻り値**: 変更があった場合は `true`、見つからなかったりファイルが存在しない場合は `false`。

## 3. エラーハンドリングと制約
- **ファイル不在**: プロセスを中断せず、`false` を返して次のファイルへ進む。
- **ヘッダー不在**: 何もせず `false` を返す。
- **標準ライブラリ**: 外部パッケージ (`npm install`) は一切使用しない。

## 4. テスト戦略 (TDD)
1.  **RED**: `tests/test_reset_skill.mjs` で、以下のケースを定義。
    - 正常系: セクションが削除され、空行がクリーンアップされる。
    - 正常系: セクションが含まれていない場合は何もしない。
    - 異常系: ファイルが存在しない場合に適切に処理する。
2.  **GREEN**: `scripts/reset_skill.mjs` を実装し、テストをパスさせる。
3.  **REFACTOR**: コードの共通化、日本語コメントの充実。
