# todo.mjs CLI 仕様再定義 デザインドキュメント

## 1. 目的

`todo.mjs` の CLI 仕様を、人間と AI の両方にとって直感的で、特に「入れ子タスク（親子関係のあるタスク）」をスムーズに扱えるように再定義する。

## 2. 課題

現状の `todo.mjs` はタスクの階層構造をサポートしておらず、すべてがフラットなリストとして扱われる。これにより、以下の問題が発生する。

-   **排他的な実行中ステータス**: 親タスクが実行中 `[/]` のときに、その子タスクを `start` できない。
-   **不自然な回避策**: 上記制約を回避するため、ユーザーは親タスクを一時的に `[x]` (完了) にするなどの不自然な操作を強いられる。

## 3. アプローチ

**インデントによる親子関係の導入**

Markdown のインデント（スペース2つまたは4つ）を解釈し、タスクの親子関係を表現する。ファイル上の見た目の階層と、スクリプト内部のデータ構造を一致させることで、直感的な操作を実現する。

## 4. デザイン詳細

### 4.1. データ構造と基本ロジック

-   **内部表現**: 各タスク行を「インデントレベル」「状態」「テキスト」を持つオブジェクトに変換し、木構造として扱う。
-   **`add <task>`**: 常にインデントなし（レベル0）でタスクを追加する。
-   **`add --child <task>`**: 現在実行中のタスクの子として、インデントを1つ下げてタスクを追加する。実行中のタスクがない場合はエラー。

### 4.2. 状態更新ロジック

-   **`start <pattern>`**:
    -   **ルール1**: 実行中のタスクがない場合、パターンに一致した最初の未完了タスクを `[/]` にする。
    -   **ルール2**: 実行中の親タスクがある場合、その**子孫**の中からパターンに一致する未完了タスクを `[/]` にできる（複数タスクの同時実行中状態を許容）。
    -   **ルール3**: 実行中のタスクと親子関係にないタスクを `start` しようとした場合は、「別の**系統**のタスクが実行中です」というエラーを返す。（"ブランチ"という言葉の混乱を避けるため"系統"を使用）
-   **`done`**:
    -   最も深い階層で実行中 `[/]` になっているタスクを完了 `[x]` にする。
    -   あるタスクの子がすべて完了 `[x]` している状態で `done` を実行すると、その親タスクが `[x]` になる。

## 5. 技術的実現性

Node.js の標準API（`fs` と文字列操作）のみで実装可能。行頭のスペース数をカウントし、正規表現でタスク状態を解析することで、外部ライブラリに依存せず実現できる。
