# デザインドキュメント: YAML Flow Sequence 対応 (scripts/gemini_sub.mjs)

- **作成日**: 2026-03-17
- **ステータス**: 承認済み
- **目的**: Gemini CLI のサブセッション管理ツール `gemini_sub.mjs` の YAML パーサーを拡張し、Flow Sequence (`[...]`) 形式のリストをサポートすることで、スキルの指定などの記述を柔軟にする。

## 1. 背景と動機
現在の自作 YAML パーサーは、ブロック形式（`- item`）のリストのみをサポートしている。しかし、`required_skills: ["using-superpowers"]` のように、一行でリストを記述したいケースが多く、これに対応することで利便性と他ツールとの互換性を向上させる。

## 2. 設計仕様

### 2.1 パースロジック
1.  **判定**: 行の `val`（値の部分）が `[` で始まり `]` で終わるか確認する。
2.  **剥離**: `[` と `]` を除去し、中の文字列を取り出す。
3.  **分割**: カンマ `,` で分割し、一時的なリストを作成する。
4.  **洗浄**: リストの各要素に対し、以下の処理を行う：
    - 前後の空白を削除 (`trim`)。
    - 一対のクォート（`"` または `'`）があれば除去 (`removeQuotes`)。
    - 再度 `trim` を行い、クォート内にあった空白も除去。
5.  **抽出**: 空文字列（`, ,` の間など）を除外して、最終的な配列を完成させる。

### 2.2 制限事項
- ネスト（入れ子）構造や、値の中にカンマを含む複雑な構造は非サポート（フラットな文字列リストのみ）。

## 3. テスト計画
- **テストファイル**: `tests/test_gemini_sub.mjs`
- **テストケース**:
    - `required_skills: ["a", "b"]` (ダブルクォート)
    - `required_skills: ['a', 'b']` (シングルクォート)
    - `required_skills: [a, b]` (クォートなし)
    - `required_skills: [ a , b ]` (空白あり)
    - `required_skills: [a,,b]` (空要素あり)
- **期待結果**: いずれの場合も `['a', 'b']` という配列として正しくパースされること。

## 4. [イメージ] (データフロー図解)
```text
[入力行] "required_skills: [ \"skill-a\" , ' skill-b ' ]"
  |
  +-- (1) 判定: "[" で始まり "]" で終わる -> [YES]
  |
  +-- (2) 剥離: " \"skill-a\" , ' skill-b ' " (大枠の [ ] を除去)
  |
  +-- (3) 分割: [ " \"skill-a\" ", " ' skill-b ' " ] (カンマで2つに分ける)
  |
  +-- (4) 洗浄:
  |       - " \"skill-a\" " -> trim -> "\"skill-a\"" -> removeQuotes -> "skill-a"
  |       - " ' skill-b ' " -> trim -> "' skill-b '" -> removeQuotes -> " skill-b " -> trim -> "skill-b"
  |
  +-- (5) 抽出: [ "skill-a", "skill-b" ] (空要素はないのでそのまま配列化)
  |
[出力データ] { required_skills: ["skill-a", "skill-b"] }
```
