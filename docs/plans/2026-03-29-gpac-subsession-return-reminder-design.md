# デザインドキュメント：GPAC サブセッション帰還リマインダー

## 1. 背景と目的
GPAC（Gemini Peer-Agent Coordination）プロトコルにおいて、サブセッションへ遷移したユーザーが作業に没頭し、完了後に親セッションへ戻る（`import`）のを忘れてしまうという課題が発生した。
本デザインは、人間には「視覚的な印象」を、AIには「論理的な制約」を与えることで、この没入による帰還忘れを物理的・心理的に防止することを目的とする。

## 2. アプローチ：ハイブリッド・リマインダー

### 2.1. 人間向け：コミカルな視覚的強調 (Visual Highlight)
親セッションで `spawn` を実行した際、コンソールに目立つ AA とカラー出力を表示する。

#### 表示イメージ (ANSI カラー適用)
- 枠線・キャラクター：シアン (`\x1b[36m`)
- 強調文字（【親セッション】）：イエロー (`\x1b[33m`) + 太字 (`\x1b[1m`)

```text
  ┌──────────────────────────────────────────┐
  │  サブセッションへ 行ってらっしゃい！     │
  │  終わったら【親セッション】へ戻るのを    │
  │  忘れないでね！待ってるよ！              │
  └─┬───────────────────────────────────────┘
    │
    │   (\__/)
    │   ( •ω•)  ＜ 迷子にならないでね！
    │   /    \
```

### 2.2. AI向け：帰還義務のプロンプト注入 (Prompt Injection)
サブセッションの起動ペイロード（`gemini` コマンドに渡すプロンプト）を拡張し、AI自身に「帰還のガイド役」としての任務を課す。

#### 注入プロンプト案
```text
GPACプロトコル：任務を定義しました。任務内容を確認するために 'node scripts/gemini_sub.mjs show-task ${taskId}' を実行してください。

【サブセッションの重要制約】
1. あなたは一時的なサブセッションです。任務完了後は速やかに 'report' を行うよう人間に促し、親セッションへの帰還（import）を誘導してください。
2. 作業が長引く場合は、定期的に「この作業はサブセッションで行っている」ことを人間にリマインドし、没入を防いでください。
```

## 3. 実装仕様

### 3.1. 変更対象ファイル
- `scripts/gemini_sub.mjs`

### 3.2. 主要な変更点
- `createPayload(workDir, taskId)`: プロンプト文字列を拡張し、制約事項を追加する。
- `launchSession(sessionId, taskPath, workDir, launcherMode)`: `console.log` を使用して、AA とカラー付きメッセージを出力するロジックを追加する。

## 4. テストと検証
1.  **表示確認**: `node scripts/gemini_sub.mjs spawn task_draft.md` を実行し、AA が意図した色とレイアウトで表示されるか確認する。
2.  **プロンプト確認**: 生成されたペイロード内のプロンプトに制約が含まれているか確認する。
3.  **AIの挙動確認**: 実際にサブセッションを起動し、AIが「自分はサブセッションである」ことを認識し、完了時に誘導を行うか試行する。
