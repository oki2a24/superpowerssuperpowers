# デザインドキュメント: superpowerSSuperpowers のフラッグシップ化

## 1. 背景と目的
現在の `superpowerssuperpowers` は、強力な「移植スキル」と、独自開発の「知能・状態管理スキル（オリジナル）」が混在している。これらを統合パッケージとして整備し、URLインストール一つで「最高峰の開発体験とエージェント知能」を提供できる状態にする。

## 2. 変更の概要
### 2.1 スキルの「昇格」 (Promotion)
`.gemini/skills/` に隔離されていたオリジナルスキルを、正規の `skills/` ディレクトリへ移動し、公開対象とする。
- 対象: `roadmap-management`, `session-coordination`, `session-handoff`, `session-retrospective`

### 2.2 汎用性の確保 (慎重な合意形成プロセス)
移動したスキルが特定のプロジェクトディレクトリに依存しないよう修正するが、**以下の点についてユーザーに具体例を交えて説明し、納得を得てから実行する。**
- **パスの動的解決**: URLインストール先（任意のディレクトリ）の `.gemini/` と、システム共通の `~/.gemini/` の役割分担。
- **具体例**: `session-handoff` が「どこに」handoffファイルを書き出すべきか（プロジェクト内か、それともグローバルか）といった設計判断。
- **混乱の防止**: 人間のユーザーが「今、どこに知見が溜まっているのか」を迷わないための、明確なディレクトリ規律の定義。

### 2.3 ブランド化 (Branding) の詳細戦略
サブセッションでの実行を見据え、本セッションで打ち出した方針を詳細に記録する。
- **「究極のスイート」コンセプト**: 開発スキル（移植版）と知能スキル（オリジナル）の二軸構成。
  - **軸1: 開発支援 (Ported Superpowers)**: TDD、デバッグ、Git連携など。これらは「今日、コードを正しく書く」ための道具。
  - **軸2: 知能管理 (Original Intelligence)**: `observation-distiller`, `session-handoff` など。これらは「明日、エージェントをより賢くする」ための資産。
- **具体的な訴求例**:
  - 「エージェントが過去の失敗から学ぶようになる。それが `observation-distiller` です。」
  - 「コンテキストウィンドウの飽和によるやり取りの崩壊を防ぎ、必要な文脈だけを次へ繋ぐ。それが `session-handoff` です。」
- **ドキュメントのトーン**: プロフェッショナルかつ「エージェントとの共鳴」を重視した、次世代の開発スタイルを提案するトーン。
