# Design: brainstorming スキルアップデート (Upstream v1.7.0 準拠)

## 概要
`obra/superpowers` の `brainstorming` スキルに含まれる重要な改善（厳格なゲート、詳細なチェックリスト、設計原則、大規模プロジェクトの分解ロジック）を反映する。

## 成功条件 (Success Criteria)

### 1. 厳格なデザイン優先原則 (HARD-GATE & Anti-Pattern)
- [ ] 承認前の実装スキル呼び出し、コード作成、プロジェクト雛形作成の厳禁を明文化する。
- [ ] 「単純すぎてデザイン不要」という思い込みを排除するためのセクションを追加する。
- [ ] 承認を得るまではいかなる実装アクションも取らないことをエージェントに徹底させる。

### 2. プロセスフローの刷新 (9-Step Checklist & Mermaid)
- [ ] Upstream の 9 ステップを日本語で正確に移植する。
- [ ] ドキュメントの保存パスをプロジェクトの慣習である `docs/plans/` に合わせる。
- [ ] プロセス図を `dot` 形式から `Mermaid` 形式へ変換し、視認性を高める。

### 3. 大規模プロジェクトの分解と設計原則
- [ ] 大規模な要求を検知し、サブプロジェクトに分解するガイダンスを追加する。
- [ ] 責務の分離（Isolation）や、既存コードベースでの振る舞い（既存パターンの尊重）に関する指針を統合する。

### 4. 自己レビューと承認ゲート
- [ ] 仕様書作成後の「自己レビュー」項目（プレースホルダ、一貫性、スコープ、曖昧さのチェック）を追加する。
- [ ] 仕様書作成後、実装フェーズに進む前に必ずユーザーのレビューを仰ぐ「承認ゲート」を設置する。
- [ ] `spec-document-reviewer-prompt.md` を日本語化して移植し、エージェントがそれを指針とできるようにする。

### 5. Visual Companion プロトコル
- [ ] 視覚的な説明（モックアップ、図解）が必要な場合にのみ提案する、専用のプロトコルを追加する。
- [ ] Gemini CLI 環境でのツール有無にかかわらず、プロトコル（手順）としての整合性を保つ。
- [ ] `visual-companion.md` を日本語化して移植する。
- [ ] `scripts/` 内のツールを評価し、Gemini CLI 環境での動作または代替案（静的 HTML 出力など）を確定させる。

## 物理的検証 (Verification)
- [ ] 更新後の `SKILL.md` を `activate_skill` で読み込み、意図した通りの指示が出力されることを確認する。
- [ ] テスト用ワークツリーにおいて、単純な要求（例：「挨拶するだけのCLIアプリを作って」）に対し、HARD-GATE が発動し、デザインフローが開始されることを実証する。
