# finishing-a-development-branch アップデート デザイン

**目標**: `obra/superpowers` の最新の `finishing-a-development-branch` スキルの構造と詳細を Gemini CLI 向けの日本語環境に適応させ、信頼性と使いやすさを向上させる。

## アーキテクチャとアプローチ
- **日本語第一主義**: オリジナルの英語の内容を正確に日本語に翻訳し、かつ Gemini CLI のトーンに合わせる。
- **コードブロックの活用**: インタラクティブな手順において、実行すべきコマンドを視覚的に分かりやすくコードブロックで提示する。
- **PR テンプレートの強化**: GitHub CLI を活用した構造化されたプルリクエスト作成手順を導入する。
- **整合性の確保**: クリーンアップ論理の矛盾（Option 2 の扱い）を解消し、ブランチ削除とワークツリー削除の条件を明確にする。
- **Gemini 固有の補足**: `.gemini/tasks/` のクリーンアップなど、プロジェクト固有の利便性を維持する。

## 変更セクションの詳細

### 1. ヘッダーと開始時の宣言
- 日本語の説明文を洗練させ、`開始時の宣言` セクションを追加する。

### 2. ステップ 1 & 2 (検証とベース特定)
- `npm test` や `git merge-base` などのコマンドをコードブロックで明示し、コピー＆ペーストや実行のイメージを具体化する。

### 3. オプション提示
- 正確に 4 つの構造化されたオプションを提示することを強調する。

### 4. 実行ステップ (Option 2 PR)
- `gh pr create` コマンドにおいて、`--body` に構造化されたテンプレート（Summary, Test Plan）を埋め込む手順を追加する。

### 5. クリーンアップ論理 (Step 5)
- **Option 1 (Merge)**: ブランチ削除、ワークツリー削除。
- **Option 2 (PR)**: ブランチ保持、ワークツリー保持（マージされるまで必要となるため）。
- **Option 3 (As-is)**: ブランチ保持、ワークツリー保持。
- **Option 4 (Discard)**: ブランチ強制削除、ワークツリー削除。
- Gemini 固有の `TODO` ファイル削除は Option 1 と 4 に限定する。

### 6. 知識セクションの拡充
- 「よくある間違い」「危険な兆候」「クイックリファレンス」をオリジナルの詳細度に合わせて翻訳・追加する。

### 7. 振り返り提案 (Retrospective)
- 現在の日本語版にある「振り返り提案」セクションは、セッションの締めくくりとして有用であるため、維持しつつ、`session-retrospective` スキルへの言及を強化する。
