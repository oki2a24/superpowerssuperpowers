# 実装計画：dispatching-parallel-agents スキルのアップデート

移植元の最新の変更（2025-10-03 以降のコミット）を Gemini CLI に適応・統合します。

## 1. 変更内容の概要

移植元 (`superpowers-original`) では、エージェントへの指示内容、具体的なデバッグシナリオ、および完了後の検証手順が大幅に強化されています。これらを日本語に翻訳しつつ、Gemini CLI の推奨形式（Mermaid）に変換して統合します。

## 2. アトミックな変更タスクと成功条件

### T1: エージェント・プロンプト構造の強化
- **内容**: 移植元の「Good agent prompts are」セクションと具体的な指示例を取り込む。
- **成功条件**: イベントベースの待機への置き換え、制約事項（本番コードを最小限にするなど）が含まれていること。

### T2: 具体的な実例（シナリオ）の追加
- **内容**: 6つのテスト失敗が発生した際の具体的なディスパッチと解決プロセスの実例を追加する。
- **成功条件**: シナリオ、意思決定、ディスパッチ、結果、統合、短縮された時間の情報が含まれていること。

### T3: 検証 (Verification) と実世界への影響セクションの追加
- **内容**: エージェントから戻った後のチェックリストと、リアルワールドでのインパクトを追加する。
- **成功条件**: 「Review each summary」「Check for conflicts」などの具体的なステップが含まれていること。

### T4: 日本語化と Mermaid 形式への変換
- **内容**: 移植元の `dot` 図を `mermaid` に変換し、すべての英文を自然な日本語に翻訳する。
- **成功条件**: 既存の `SKILL.md` のトーンに合わせ、すべての英文が日本語化されていること。

## 3. 実装アプローチ

1. **セクション単位での `replace`**: 一度にすべてを書き換えず、論理的に独立したセクションごとに更新を適用する。
2. **Mermaid 優先**: 移植元は `dot` ですが、本プロジェクトの標準である `mermaid` を維持する（必要に応じて `dot` のロジックを反映）。
3. **日本語第一主義**: 単なる翻訳ではなく、Gemini CLI のコンテキスト（マルチ・ワークツリー等）に馴染む表現を選択する。

## 4. 検証戦略

1. **文法チェック**: `mermaid` 図が正しくレンダリングされるか確認（構文エラーがないか）。
2. **セルフレビュー**: 日本語の自然さと、指示の明確さを確認。
3. **コードレビュー**: `code-reviewer` エージェントによる最終ゲートの通過。
4. **実証テスト**: `/skill reload` 後に `activate_skill` で内容が正しく表示されることを確認。
