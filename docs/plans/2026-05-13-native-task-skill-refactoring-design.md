# デザインドキュメント：スキル群のネイティブタスク管理への移行

## 背景
Gemini CLI にはネイティブのタスク管理機能（`write_todos`, `tracker`等）がビルトインされています。これまでの移植スキル群は、その欠如を補うために `todo.mjs` (Markdownベース) に依存していましたが、トークン効率の向上と CLI UI との親和性を高めるため、ネイティブ機能を優先する設計へと一括リファクタリングを行います。

## 目的
- 全ての移植スキルにおいて、Gemini CLI ビルトインタスク管理を第一選択とする。
- `todo.mjs` は「セッションを跨ぐ永続化」および「緊急時のフォールバック」として温存する。
- スキル内の具体的なコマンド記述（`node scripts/todo.mjs ...`）を抽象化し、プロトコル化する。

## 提案するアプローチ：タスク管理プロトコルの抽象化
各スキルの `SKILL.md` に記載されている「具体的な `todo.mjs` の実行手順」を、以下の **「標準タスクプロトコル」** への参照に置き換えます。

### 1. 規律の統合
- `roadmap-management` および `observations/GEMINI.md` で定義した「ネイティブ・ファースト & サイレント・フォールバック」を SSOT とする。
- 各スキルは「タスク管理を行え」と指示し、その具体手法は `GEMINI.md` の規律に従うものとする。

### 2. 修正対象スキルと変更方針
以下のスキルに含まれる `todo.mjs` への直接言及を修正します。

| スキル名 | 現在の状態 | 修正方針 |
| :--- | :--- | :--- |
| `executing-plans` | `todo.mjs` で全タスクを初期化・更新 | 「タスク管理ツール」の使用に抽象化。ネイティブ優先。 |
| `test-driven-development` | RED/GREEN/REFACTOR 毎に `todo.mjs` を実行 | 各ステップの進捗をネイティブで報告。同期は任意。 |
| `systematic-debugging` | 4フェーズを `todo.mjs` に登録 | フェーズ管理をネイティブに移行。 |
| `subagent-driven-development` | サブエージェントとの進捗共有に利用 | ネイティブタスク（および Plan Mode）での共有を推奨。 |
| `writing-skills` | チェックリストを `todo.mjs` で管理 | ネイティブタスクとしての展開に切り替え。 |

### 3. トークン節約の設計
- **IDRプロトコルの回避**: ネイティブ管理では ID の確認が不要（AI が内部状態として保持）なため、`show --json` の呼び出し回数を削減できます。
- **バッチ更新の推奨**: `todo.mjs` への同期は、一連の作業が完了した「チェックポイント」でのみ行うように指示を書き換えます。

## 懸念事項と対策
- **懸念**: ネイティブ機能はセッションを跨いで永続化されない。
- **対策**: `session-retrospective` や `session-handoff` などの終了処理スキルに、「現在のネイティブタスクを `todo.mjs` (TODO.md) にスナップショット保存する」指示を必須化する。

## 次のステップ
1. 本デザインドキュメントの承認を得る。

3. `session-retrospective` に「スナップショット保存」の指示を追加し、フォールバック体制を完成させる。
