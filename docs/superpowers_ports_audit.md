# Superpowers 移植監査とメンテナンス指針 (2026-04-01)

このドキュメントは、`obra/superpowers` から Gemini CLI への移植状況を記録し、将来のアップデート追随時の判断材料とするためのものです。

## 1. 移植調査の要約 (Session Summary)

2026年4月1日のセッションにおいて、移植元 (`superpowers-original/`) と現在のワークスペースを網羅的に比較調査しました。
「スキルは全て移植済み」という初期認識から一歩踏み込み、エージェント、ドキュメント、ライブラリ、コマンド、テストの各要素について、移植の要否を判断しました。

## 2. 移植状況マトリクス (Status Matrix)

| 要素 (Directory/File) | 状況 (Status) | 判断の背景 (Reasoning) |
| :--- | :--- | :--- |
| `skills/` | **移植済み** | 全てのスキルが Gemini CLI 向けに移植され、日本語化されています。 |
| `hooks/` | **移植済み** | `session-start.sh` が `session-start.mjs` (Node.js) として移植されています。 |
| `agents/code-reviewer.md` | **詳細化済み** | 移植元の英語版が現在の日本語版より詳細であったため、指示をマージしました。 |
| `docs/testing.md` | **日本語移植済み** | テストの設計思想や検証ポイントが有用であるため、日本語翻訳して移植。ファイル名は維持。 |
| `lib/skills-core.js` | **移植見送り** | スキル検索や Git 更新確認のロジック。Gemini CLI 本体と機能重複し、保守性が下がるため。 |
| `commands/*.md` | **移植見送り** | Claude Code のスラッシュコマンド定義。Gemini スキルと機能重複しているため。 |
| `tests/` (移植元) | **移植見送り** | Claude 固有の `.jsonl` ログ解析に依存しており、Gemini 環境では動作不能なため。 |

## 3. 重要な判断の記録 (Key Decisions)

### エージェントの「詳細版」へのアップデート
`agents/code-reviewer.md` は、単なる「レビューしてください」という指示から、計画との整合性、アーキテクチャの評価、通信プロトコルを含む「シニアコードレビュアー」のペルソナへと強化されました。これは、スキルの品質を担保する上で不可欠な要素です。

### `docs/testing.md` のファイル名維持
移植元のドキュメントを `TESTING_REFERENCE.md` などの別名にせず、元の `testing.md` という名称を維持しました。これは、将来 `superpowers-original/docs/testing.md` が更新された際に、`git diff` や `meld` 等のツールで差分を容易に比較・同期できるようにするためです。

### 動作を妨げるロジックの排除
`lib/skills-core.js` の Git アップデート確認機能などは、便利そうに見えますが、サブモジュールとの関係やネットワークコストを考慮し、Gemini CLI の「シンプルで軽快な動作」を損なうと判断して除外しました。

## 4. アップデート追随ガイド (Maintenance Guide)

`superpowers-original` (サブモジュール) が更新された際は、以下の手順で同期を検討してください。

1. **スキルの同期**: 
   - `skills/<skill_name>/SKILL.md` を比較。
   - スキル本体は純粋な移植状態（SSOT）を維持し、移植元の更新をそのまま反映させる。
   - 本プロジェクト固有の改善や知見は、`observations/` ディレクトリの対応する層（L2/L4）にある知見ファイルへ分離・マージする。
2. **ドキュメントの同期**: 
   - `docs/testing.md` を比較。
   - 翻訳済みの日本語構造を維持しつつ、新しい検証ポイント（アサーション）を追記する。
3. **エージェントの同期**: 
   - `agents/code-reviewer.md` を比較。
   - レビュー基準（プロンプト）の強化があれば日本語でマージする。
