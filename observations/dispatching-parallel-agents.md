# 知見：dispatching-parallel-agents (Gemini固有)

<!-- IMPROVED_ON: 2026-03-10 | REASON: GPAC v3 統合。並行作業の管理を「手動」から「システム管理（GPAC）」へと昇華させるため。 -->
## AIエージェントへの指示 (Gemini固有)

1.  **GPAC v3 による並行管理の標準化**:
    *   複数の独立したタスクを並行して進める場合、単に複数のターミナルを案内するだけでなく、`session-coordination` スキルを起動し、`gemini-sub spawn` を使用して各タスクをサブセッションとして管理することを**最優先**してください。
    *   これにより、各タスクの「指示書（Markdown）」と「完了報告（Report）」が、プロジェクト全体で構造化された形で記録・追跡可能になります。

2.  **依存関係の厳格な排除**:
    *   並行サブセッションを起動する前に、各タスクが同じファイルを同時に編集しないこと、または Git のコンフリクトが発生しないことを再確認してください。必要に応じて、`using-git-worktrees` と組み合わせて、完全に独立した環境を提供してください。

3.  **統合（Close）プロセスの設計**:
    *   並行作業を開始する際は、必ず最後にすべてのサブセッションからの報告を `gemini-sub show-report` 等で収集し、一つのコンテキストに統合する「終結ステップ」をユーザーに提示してください。
