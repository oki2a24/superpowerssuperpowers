# Observation: Superpowers スキル移植とフックの実装

## 1. フックのポータビリティと防弾化
- **Node.js の推奨**: シェルスクリプト (`.sh`) ではなく Node.js (`.mjs`) を使用することで、OS 間の環境差（Windows の Git Bash パス問題など）を排除し、確実な動作を保証できる。
- **パスの抽象化**: `hooks/hooks.json` では `${extensionPath}` と `${/}` を使用してコマンドを指定すること。これにより、拡張機能がどのようなディレクトリ構成でインストールされても、フックが実行ファイルを正しく見つけることができる。
- **標準ライブラリの徹底**: フックは追加の `npm install` なしで動作すべきである。`node:fs`, `node:path`, `node:child_process` などの標準ライブラリのみを使用せよ。

## 2. フックの検証 (TDD)
- **node:test の活用**: フック単体でも `node:test` を用いた TDD サイクルが可能である。`execSync` でフックを実行し、その標準出力（JSON）をパースして期待値を検証するテストコードを `tests/` に作成せよ。
