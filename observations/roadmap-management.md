# 知見： roadmap-management (L2)

## 概要
`scripts/todo.mjs` を用いた、状態遷移エラーのない確実なロードマップ管理。

## 知見 (Observations)
- **ID ベースの操作を優先せよ**: 曖昧さを排除するため、`start` 等のコマンドでは必ず数値 ID を使用せよ。
- **操作前の `show` 実行**: タスク ID と現在の状態を物理的に確認するため、アクションの前には必ず `node scripts/todo.mjs show` を実行せよ。
- **AI ガイドの参照**: `scripts/TODO_GUIDE.md` が存在する場合、作業開始前に必ず一読し、その規律を脳内にロードせよ。
- **自動サスペンドの活用**: 別の系統のタスクを `start` すると、現在のアクティブなタスクは自動的に `[ ]` (suspend) に戻る。手動での停止エラーを恐れず、コンテキストスイッチを行え。
- **回帰テストの実施**: `todo.mjs` 自体、あるいはその管理下にあるコードを修正した後は、必ず `find tests -name "test_todo_*.mjs" -exec node {} \;` でデグレードがないことを物理的に証明せよ。
