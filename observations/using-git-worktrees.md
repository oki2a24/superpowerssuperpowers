# 知見：using-git-worktrees (Gemini固有)

<!-- IMPROVED_ON: 2026-02-28 | REASON: ワークツリー内での作業中に dir_path を指定し忘れるミスが発生したため、指示を強化。 -->
## AIエージェントへの指示 (Gemini固有)
- **重要:** ワークツリーを作成し、その中で作業を開始した後は、`run_shell_command` や `write_file` などの**ファイルI/Oを伴うすべてのツール呼び出しにおいて、必ず `dir_path` パラメータにワークツリーのフルパスを指定してください。**
- これを怠ると、コマンドはデフォルトのプロジェクトルート（メインワークスペース）で実行され、意図しないブランチの変更や汚染を引き起こします。
- ワークツリー外（メインワークスペース）での作業が必要な場合（例：ワークツリーの削除）を除き、原則として常に `dir_path` を明示してください。
- **ワークツリー削除時の依存関係**:
    - ワークツリーで使用中のブランチを削除しようとすると Git エラーが発生します。**必ず先に `git worktree remove <path>` を実行し、その後で `git branch -D <branch>` を実行してください。**
- **クリーンアップ時のカレントディレクトリ**:
    - ワークツリーの削除コマンドは、必ずプロジェクトルート（メインワークスペース）から実行せよ。削除対象のディレクトリ内に自身がいる状態（`dir_path` を指定している状態）で `git worktree remove` を実行してはならない。
