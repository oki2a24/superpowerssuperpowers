# todo.mjs AI 専用操作ガイド

## AI エージェントのための核心的規律

状態遷移エラーを回避し、確実なタスク管理を行うため、すべての AI エージェントは以下のルールを厳守すること：

1.  **アクションの前に必ず `show` を実行せよ**: タスクを開始・完了する前に、必ず `node scripts/todo.mjs show` を実行して現在のステータスと正しい **タスク ID** を確認すること。
2.  **ID ベースのコマンドを優先せよ**: 曖昧さを排除するため、パターンではなくタスク ID（例: `1`, `2`）を使用すること。
    - `node scripts/todo.mjs start 1`
    - `node scripts/todo.mjs done` (現在アクティブなタスクを完了にする)
3.  **自動サスペンド (Auto-suspend) を理解せよ**: 新しいタスクを開始すると、前のアクティブなタスクは自動的に「未着手」に戻される（親子関係にある場合を除く）。手動で停止する必要はない。
4.  **変更後はすべてのテストを確認せよ**: コードを修正した後は、必ず `find tests -name "test_todo_*.mjs" -exec node {} \;` を実行し、デグレードがないことを確認すること。

## コマンドリファレンス

- `init <タイトル>`: 現在のブランチに対応する新しい TODO を初期化。
- `add <タスク内容> [--child]`: タスクを追加。`--child` を使うと現在のアクティブなタスクの子タスクとして作成される。
- `start <ID|パターン>`: タスクを開始。数値 ID の使用を強く推奨。
- `done`: 最も深い階層のアクティブなタスクを完了にする。
- `show`: ID 付きのダッシュボードを表示。
