# todo.mjs AI 専用操作ガイド (v2)

## AI エージェントのための核心的規律

状態遷移エラーを回避し、確実なタスク管理を行うため、すべての AI エージェントは以下の **IDR プロトコル** を厳守すること：

1.  **Inspect (現状確認)**: アクションの前に必ず `node scripts/todo.mjs show --json` を実行し、最新のステータスと正しい **数値タスク ID** を取得せよ。自身のメモリにある古い ID を信用してはならない。
2.  **Do (正確な実行)**: 
    - 曖昧さを排除するため、パターンではなく数値 ID を使用せよ。
    - フラグ（`--child` 等）の配置は問わないが、引数は可能な限りクォートせよ。
3.  **Read-back (状態検証)**: 実行直後に再び `show --json` を実行し、期待通りにステータス（`status`）が更新されたことを物理的に確認せよ。

## コマンドリファレンス

- `init <タイトル>`: 新しい TODO を初期化。
- `add <内容> [--child]`: タスクを追加。フラグの位置は不問。
- `start <ID|パターン>`: タスクを開始。数値 ID を推奨。
- `done`: 最も深い階層のアクティブなタスクを完了にする。
- `show [--json]`: 進捗を表示。AI は `--json` を使用して確実にパースすること。

## エラーハンドリング
- エラーはすべて `stderr` に出力される。
- 成功時は `stdout` に結果（JSON またはダッシュボード）が出力される。
