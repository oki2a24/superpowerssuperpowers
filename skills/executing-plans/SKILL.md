# 計画の実行

## 概要

書かれた実装計画をロードし、批判的にレビューし、タスクをバッチで実行し、バッチ間でレビューのために報告します。

**コア原則:** アーキテクトレビューのためのチェックポイントを持つバッチ実行。

## プロセス

### 1. 計画のロードとレビュー

*   **AIエージェントへの指示**: ユーザーに「実行する計画ファイル（Markdown形式）へのパス」を尋ねてください。取得後、そのファイルを読み込み、内容をユーザーに提示し、問題がないか確認してください。
*   読み込んだ計画ファイルを批判的にレビューし、計画に関する質問や懸念点があれば特定します。
*   懸念点がある場合、作業を開始する前にユーザーに質問します。
*   懸念がない場合、計画ファイルからTodoリストを抽出します。`run_shell_command` を使い、`node scripts/todo.mjs init <タイトル>` でToDoファイルを初期化した後、各タスクを `node scripts/todo.mjs add <タスク内容>` で登録してください。

### 2. バッチ実行

*   **デフォルト**: 最初の3タスクをバッチとして実行します。
*   各タスクについて:
    1.  `run_shell_command` を使い、`node scripts/todo.mjs start <検索パターン>` でタスクを `[/]` (実行中) にマークします。
    2.  計画に指定された各ステップを正確に実行します（計画は一口サイズのステップで構成されています）。
    3.  指定された検証を実行します。
    4.  `run_shell_command` を使い、`node scripts/todo.mjs done` でタスクを `[x]` (完了) にマークします。

### 3. レポート

バッチが完了したら、以下を報告します。

*   実装された内容を示します。
*   検証の出力を示します。
*   「フィードバックの準備ができました。」とユーザーに伝えます。

### 4. 続行

フィードバックに基づき、以下を行います。

*   必要に応じて変更を適用します。
*   次のバッチを実行します。
*   完了するまで繰り返します。

### 5. 開発の完了

すべてのタスクが完了し、検証された後、以下を行います。

*   **AIエージェントへの指示**: 移植された`finishing-a-development-branch`スキルを`activate_skill`で起動してください。
*   そのスキルに従い、テストの検証、オプションの提示、選択したオプションの実行を行います。

## ヘルプが必要な場合の停止と質問

**以下の場合、直ちに実行を停止します。**

*   バッチの途中でブロッカーに遭遇した場合（依存関係の不足、テストの失敗、指示の不明瞭など）。
*   計画に重大なギャップがあり、開始できない場合。
*   指示が理解できない場合。
*   検証が繰り返し失敗する場合。

**推測するのではなく、明確化のために質問します。**

## 以前のステップへの再訪問

**以下の場合、レビュー（ステップ1）に戻ります。**

*   ユーザーがフィードバックに基づいて計画を更新した場合。
*   根本的なアプローチの再検討が必要な場合。

**ブロッカーを強行突破しないこと** - 停止して質問します。

## 重要な注意点

*   まず計画を批判的にレビューします。
*   計画のステップに正確に従います。
*   検証をスキップしないでください。
*   計画が指示するスキルは`activate_skill`を使用して参照します。
*   バッチ間では、報告して待機するだけです。
*   ブロッカーに遭遇したら停止し、推測しないでください。
*   **AIエージェントへの指示**: ユーザーの明示的な同意なしに、`main`ブランチで実装を開始しないでください。

## 統合

**必要なワークフロースキル:**

*   `using-git-worktrees` - 必須: 開始前に隔離されたワークスペースをセットアップします。
*   `writing-plans` - このスキルが実行する計画を作成します。
*   `finishing-a-development-branch` - すべてのタスク完了後に開発を完了します。
