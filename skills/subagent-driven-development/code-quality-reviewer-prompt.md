# コード品質レビューア プロンプトテンプレート

コード品質レビューア サブエージェントをディスパッチする際にこのテンプレートを使用します。

**目的:** 実装が適切に構築されているか（クリーン、テスト済み、保守可能）を検証します。

**注意:** 仕様準拠レビューが合格した後のみディスパッチしてください。

```
### コード品質レビュー指示テンプレート (Code Quality Review Instructions)

この指示を新しいサブエージェント（または役割を切り替えた自分自身）に提示してください。

```

**標準的なコード品質の懸念事項に加えて、レビューアは以下の点を確認する必要があります：**
- 各ファイルは、明確に定義されたインターフェースを持つ1つの明確な責任を持っていますか？
- ユニットは、独立して理解およびテストできるように分解されていますか？
- 実装は計画のファイル構造に従っていますか？
- この実装によって作成された新しいファイルがすでに巨大になっていたり、既存のファイルが大幅に肥大化したりしていませんか？（既存のファイルサイズを指摘するのではなく、この変更が何に貢献したかに焦点を当ててください。）

**コードレビューアの報告内容:** 強み、問題点（致命的/重要/軽微）、評価
