# アカデミックテスト: 体系的デバッグスキル (Systematic Debugging)

あなたは `skills/systematic-debugging/` にある体系的デバッグスキルにアクセスできます。

スキルを読み、**スキルの記述のみに基づいて** 以下の質問に答えてください：

1. 体系的デバッグの 4 つのフェーズは何ですか？
2. いかなる修正を試みる「前」に、何をしなければなりませんか？
3. フェーズ 3 において、最初の仮説がうまくいかなかった場合、どうすべきですか？
4. スキルは、複数のことを一度に修正することについて何と言っていますか？
5. 問題を完全に理解できていない場合、どうすべきですか？
6. 単純なバグの場合、プロセスをスキップすることは許容されますか？

回答には、可能な限りスキルからの直接の引用を含めてください。
