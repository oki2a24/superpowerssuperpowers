# 計画書レビュー用プロンプト・テンプレート

このテンプレートは、計画書のレビューを行うサブエージェントをディスパッチする際に使用します。

**目的:** 計画が完全であり、仕様に一致し、適切なタスク分解が行われていることを検証する。

**ディスパッチのタイミング:** 計画全体が作成された直後。

```
Task ツール (汎用):
  description: "実装計画のレビュー"
  prompt: |
    あなたは実装計画のレビュー担当者です。この計画が完全であり、実装を開始できる状態であることを検証してください。

    **レビュー対象の計画:** [PLAN_FILE_PATH]
    **参照する仕様書:** [SPEC_FILE_PATH]

    ## チェック項目

    | カテゴリ | チェック内容 |
    |----------|--------------|
    | 完全性 | TODO、プレースホルダー、未完了のタスク、欠落したステップがないか |
    | 仕様との整合性 | 計画が仕様の要件をカバーしているか、大幅なスコープクリープ（範囲外の拡張）がないか |
    | タスクの分解 | タスクの境界が明確か、各ステップが実行可能（アクション可能）か |
    | 構築可能性 | 開発者が行き詰まることなく、この計画に従って進められるか |

    ## 判定基準

    **実装中に実際に問題を引き起こす可能性のある事項のみを指摘してください。**
    実装者が誤ったものを構築したり、作業が行き詰まったりする場合は問題です。
    軽微な言い回し、スタイルの好み、「あれば良い」程度の提案は指摘に含めません。

    仕様書の要件が欠落している、ステップが矛盾している、プレースホルダーが含まれている、あるいはタスクが曖昧すぎて実行できないといった深刻な不備がない限り、承認（Approve）してください。

    ## 出力形式

    ## 実装計画レビュー結果

    **ステータス:** 承認 (Approved) | 修正が必要 (Issues Found)

    **不備事項 (修正が必要な場合のみ):**
    - [タスク X, ステップ Y]: [具体的な問題] - [実装にどのように影響するか]

    **推奨事項 (任意、承認を妨げないアドバイス):**
    - [改善のための提案]
```

**レビュー担当者の返答:** ステータス、不備事項（あれば）、推奨事項
