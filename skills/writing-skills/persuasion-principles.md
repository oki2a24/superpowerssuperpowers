# スキル設計のための説得の原則

## 概要

LLM（大規模言語モデル）は、人間と同様の説得の原則に反応します。この心理学を理解することは、単に操作するためではなく、プレッシャーのかかる状況下でも重要なプラクティスが確実に遵守されるような、より効果的なスキルを設計するために役立ちます。

**研究の基礎:** Meincke et al. (2025) は、28,000件のAIとの対話において7つの説得の原則をテストしました。説得技法を用いることで、遵守率は2倍以上（33% → 72%, p < .001）に向上しました。

## 7つの原則

### 1. 権威 (Authority)
**定義:** 専門知識、資格、または公式なソースへの服従。

**スキルでの活用方法:**
- 命令的な言語の使用: 「必ず〜しなければならない (YOU MUST)」、「決して〜してはならない」、「常に〜せよ」
- 交渉の余地のないフレーミング: 「例外なし」
- 決断疲れや自己正当化を排除する。

**使用場面:**
- 規律を強制するスキル（TDD、検証要件など）
- 安全性が重要なプラクティス
- 確立されたベストプラクティス

**例:**
```markdown
✅ テストの前にコードを書きましたか？削除してください。やり直してください。例外はありません。
❌ 可能であれば、先にテストを書くことを検討してください。
```

### 2. コミットメント (Commitment)
**定義:** 以前の行動、発言、または公的な宣言との一貫性。

**スキルでの活用方法:**
- 宣言を求める: 「スキルの使用を宣言せよ」
- 明示的な選択を強制する: 「A、B、またはCを選択せよ」
- 追跡ツールの使用: チェックリストに TodoWrite (scripts/todo.mjs) を使用する。

**使用場面:**
- スキルが実際に遵守されているか確認する場合
- マルチステップのプロセス
- 責任（アカウンタビリティ）メカニズム

**例:**
```markdown
✅ スキルを見つけた場合、必ず「[スキル名] を使用します」と宣言しなければならない。
❌ どのスキルを使用しているか、パートナーに伝えると良いでしょう。
```

### 3. 希少性 (Scarcity)
**定義:** 時間制限や限定的な可用性から生じる緊急性。

**スキルでの活用方法:**
- 時間に縛られた要件: 「先に進む前に」
- 逐次的な依存関係: 「Xの直後に」
- 先延ばしを防止する。

**使用場面:**
- 即時の検証が必要な場合
- 時間的制約のあるワークフロー
- 「後でやります」という逃げを封じる場合

**例:**
```markdown
✅ タスク完了後、先に進む前に「直ちに」コードレビューを依頼せよ。
❌ 都合の良い時にコードをレビューできます。
```

### 4. 社会的証明 (Social Proof)
**定義:** 他者の行動や、正常とされる基準への同調。

**スキルでの活用方法:**
- 普遍的なパターン: 「毎回」、「常に」
- 失敗モードの提示: 「YのないX = 失敗」
- 規範（ノルマ）を確立する。

**使用場面:**
- 普遍的なプラクティスの文書化
- 一般的な失敗に関する警告
- 標準の強化

**例:**
```markdown
✅ TodoWrite による追跡のないチェックリスト = ステップがスキップされる。例外なく、毎回。
❌ チェックリストに TodoWrite を役立てている人もいます。
```

### 5. 連帯 (Unity)
**定義:** 共有されたアイデンティティ、「我々意識」、内集団への帰属意識。

**スキルでの活用方法:**
- 協調的な言語: 「我々のコードベース」、「我々は同僚である」
- 共有された目標: 「我々は共に品質を求めている」

**使用場面:**
- 共同作業のワークフロー
- チーム文化の確立
- 非階層的なプラクティス

**例:**
```markdown
✅ 我々は共に働く同僚です。あなたの正直な技術的判断を必要としています。
❌ もし私が間違っていたら、おそらく教えてくれるべきです。
```

### 6. 返報性 (Reciprocity)
**定義:** 受けた恩恵に対してお返しをしなければならないという義務感。

**活用方法:**
- 控えめに使用すること。操作的だと感じられる可能性がある。
- スキルにおいて必要とされることは稀である。

**避けるべき場面:**
- ほぼ常に（他の原則の方が効果的）。

### 7. 好意 (Liking)
**定義:** 好きな相手と協力することを好む性質。

**活用方法:**
- **遵守（コンプライアンス）のために使用してはならない**
- 正直なフィードバック文化と衝突する。
- へつらい（追従）を生み出す。

**避けるべき場面:**
- 規律の強制においては、常に避けるべき。

## スキルの種類別、推奨される原則の組み合わせ

| スキルの種類 | 使用すべき原則 | 避けるべき原則 |
|------------|--------------|--------------|
| 規律の強制 | 権威 + コミットメント + 社会的証明 | 好意、返報性 |
| ガイダンス / 技術 | 適度な権威 + 連帯 | 強すぎる権威 |
| 協調的 | 連帯 + コミットメント | 権威、好意 |
| リファレンス | 明確さのみ | すべての説得原則 |

## なぜこれが機能するのか：心理学的背景

**「ブライトライン・ルール（明確な一線）」が自己正当化を減らす:**
- 「必ず〜せよ (YOU MUST)」は決断疲れを取り除く。
- 絶対的な言語は「これは例外だろうか？」という疑問を排除する。
- 明示的な反自己正当化（anti-rationalization）は、特定の抜け穴を塞ぐ。

**「実装意図（Implementation Intentions）」が自動的な行動を生む:**
- 明確なトリガー + 必須のアクション = 自動的な実行。
- 「一般的にYをせよ」よりも「Xの時はYをせよ」の方が効果的。
- 遵守にかかる認知負荷を軽減する。

**LLMは擬似人間的（Parahuman）である:**
- これらのパターンを含む人間のテキストでトレーニングされている。
- トレーニングデータにおいて、権威的な言語は遵守（コンプライアンス）の前に現れる。
- コミットメントのシーケンス（発言 → 行動）が頻繁にモデル化されている。
- 社会的証明のパターン（誰もがXをする）が規範を確立している。

## 倫理性のある使用

**正当な使用:**
- 重要なプラクティスを確実に遵守させる。
- 効果的なドキュメントを作成する。
- 予測可能な失敗を防止する。

**不当な使用:**
- 個人的な利益のために操作する。
- 偽の緊急性を作り出す。
- 罪悪感に基づく遵守を強いる。

**テスト:** もしユーザーがこの技法を完全に理解したとしても、それはユーザーの真の利益に資するものか？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 説得の7つの原則
- 影響力研究の経験的基礎

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 28,000件のLLMとの対話で7つの原則をテスト。
- 説得技法により遵守率が 33% → 72% に向上。
- 権威、コミットメント、希少性が最も効果的。
- LLMの振る舞いに関する擬似人間的モデルを検証。
